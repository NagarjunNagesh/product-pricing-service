Criterios de evaluación de la prueba,
Para la valoración de esta prueba se tendrán en cuenta, además de todo lo comentado en el 
enunciado, los siguientes puntos:
• Entregar la prueba en un repositorio
• Arquitectura Hexagonal
• Eficiencia de la extracción de datos
• Pruebas de integración solicitadas en el enunciado
• Claridad de código
• Endpoint GET con buenas prácticas 
• Inicializar con los datos del ejemplo al arrancar la aplicación - H2
• Readme
• SOLID
• API REST
• Eficiencia
• Testing
• Claridad de código
• Control de versiones
• Configuración
• Devolver único resultado
• Parámetros de entrada: fecha, id de cadena e id de producto
• Retorno: id producto, id cadena, tarifa a aplicar, fechas y precio
